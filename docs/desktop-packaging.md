# Desktop Packaging

This project now packages the Kotlin backend into the Tauri app resources so the desktop shell can start it automatically.

## What gets bundled

- Frontend: Tauri + Vue desktop UI
- Backend: Gradle `installDist` output copied into `desktop/src-tauri/resources/backend`
- Runtime: a trimmed JRE generated by `jlink` and copied into `desktop/src-tauri/resources/jre`
- Startup: Tauri launches `backend/bin/dmanager-backend` on Unix or `backend/bin/dmanager-backend.bat` on Windows, with `JAVA_HOME` pointed at the bundled runtime

## Local prepare step

The desktop package scripts now run the backend prepare step automatically:

```bash
npm --prefix desktop run tauri:dev
npm --prefix desktop run tauri:build
```

The prepare step runs the Gradle task below:

```bash
backend/gradlew syncTauriResources
```

## Required environment

Backend packaging needs a working JDK because Gradle builds the backend distribution and runs `jlink`.

Recommended variables on Windows:

```powershell
$env:JAVA_HOME="C:\Users\zrx15\.jdks\ms-17.0.18"
$env:Path="$env:JAVA_HOME\bin;$env:Path"
$env:GRADLE_USER_HOME="C:\Core\Code\DockerManager\.gradle-home"
```

## Cross-platform outputs

Tauri is configured to emit native bundles per platform:

- Windows: `msi`
- macOS: `dmg`, `app`
- Linux: `appimage`, `deb`, `rpm`

Build each target on its matching OS in CI for the most reliable result.

## GitHub Actions

The repository now includes a dedicated desktop packaging workflow:

- File: `.github/workflows/desktop-release.yml`
- Triggers: manual run and pushed tags matching `v*`
- Artifacts:
  - Windows: `msi`
  - macOS: `dmg`, `.app.tar.gz`
  - Linux: `AppImage`, `deb`, `rpm`

## Current limitation

The backend is now self-contained with a bundled Java runtime, but it is still a JVM app. If you later want even smaller packages and faster cold start, the next step would be a native executable build.
